# Zip

CatchAdmin 提供一个 Zip/phar 压缩的功能，主要用于模块的打包和解压，主要为了之后模块的生态作准备，当然开发者有压缩需求也是可以使用的。 可以使用门面 Catcher\Facade\Zipper 操作。

# 简单的例子

Zipper::make('test.zip')->add('test.txt')->close();

# 主要的一些方法

# make($pathToFile)

创建或者打开一个 Zip 文件，如果文件不存在的话，他会创建一个新的，并且会返回一个 Zipper 对象用于后续的操作

# add($files/folder)

这个方法可以添加文件(数组)或者这个文件夹，如果是文件夹，那么文件夹内的文件会被添加进去。但是这里要注意的是，他并不会添加文件夹到 zip 包内，而仅仅是添加文件夹的文件到包内。

# addString($filename, $content)

通过将名称和内容指定为字符串，将单个文件添加到 zip

# remove($file/s)

从 zip 包内移除文件

# folder($folder)

如果需要往 Zip 添加文件夹，并且向文件夹内添加文件，那么就必须使用该方法，下面是一个简单的例子

Zipper::make('test.zip')->folder('first')->add('index.php');

那么 test.zip 在解压之后就会包含一个 test 文件夹以及 test/index.php 文件

# listFiles($regexFilter = null)

列出 zip 内的所有文件，那然也可以使用正则过滤，比如列出 zip 内的所有 .php 文件

Zipper::make('test.zip')->listFiles('/\.php$/i');

# close()

使用结束后调用，关闭资源

# extractTo($path)

将 Zip 的内容解压到指定的位置，简单的案例,解压到当前目录

Zipper::make('test.zip')->extractTo('./');

当然也可以选择 Zip 内的某一个文件夹。例如

Zipper::make('test.zip')->folder('first')->extractTo('./');

# extractMatchingRegex($path, $regex)

可以使用正则过滤需要的文件，在解压。例如：只需要解压 .php 文件

Zipper::make('test.zip')->folder('first')->extractMatchingRegex('./', '/\.php$/i');

# 模块打包完成代码

最后来看下，CatchAdmin 模块打包的完整代码，非常简单。

// 模块的路径
$modulePath = $path ? $path . DIRECTORY_SEPARATOR . $moduleName : CatchAdmin::getModulePath($moduleName);

// 默认 zip 将打包在 storage 目录下
$zipper =  Zipper::make(storage_path() . DIRECTORY_SEPARATOR . $moduleName . '.zip');

foreach (File::allFiles($modulePath) as $file) {
    if ($file->getRelativePath()) {
        // 将目录添加到 zip 内，并且添加文件
        $zipper->folder($file->getRelativePath())->add($file->getPathname());
    } else {
        // 这是添加根目录下文件
        $zipper->folder('')->add($file->getPathname());
    }
}

$zipper->close();
